# SPEC.md — 拡散モデルによるシンボリック回帰

作成日: 2026-07-04（フェーズ2成果物）
ステータス: **ドラフト（ユーザー承認待ち）**

## 0. 背景と目的

観測データ (X, y) から、それを説明する明示的な数式（例: `y = x0^2 + 2.5*cos(x3) - x7`）を自動発見する。
数式を**トークン列**として表現し、**離散拡散モデル**の逆過程（デノイジング）によって、データに条件付けられた数式を生成する。

- **想定用途**: 物理法則の発見（クリーンな合成データを対象）
- **最終成果物**: 動くプロトタイプ ＋ ベンチマーク結果（ベースライン比較付き）
- **評価の重点**: ①記号的一致（式の復元率）②式の簡潔さ・解釈性

### 2層構成の前提（フェーズ1で合意）

実行環境は CPU のみのため、以下の2層で進める。

| 層 | 内容 |
|---|---|
| **設計層（本仕様書）** | 最大5変数・標準演算子集合のフルスケール設計を記述 |
| **プロトタイプ層** | 同一コードベースの縮小構成（1〜3変数・小型モデル・小データ）で CPU 上の end-to-end 動作とベンチマークを実施 |

## 1. スコープ

### やること

- 数式スケルトン（定数プレースホルダ `C` 入り）の前置記法トークン列に対する**マスク型（吸収状態）離散拡散モデル**の実装
- (X, y) の**順序不変エンコーダ**（Set Transformer 風）による条件付け
- 生成後の **BFGS による定数フィッティング**（`scipy.optimize.minimize`）
- 合成データによる事前学習（式の事前分布からのサンプリング → 評価点生成）
- best-of-k サンプリングと「データ適合度＋複雑度」による候補選択
- 評価: 記号的一致率（SymPy 等価判定）、R²、式の複雑度、対ベースライン比較（gplearn、線形回帰+Lasso 必須／PySR は導入できれば）
- pytest によるユニットテスト、README.md、EXPERIMENTS.md

### やらないこと

- ノイズ付きデータへのロバスト性（クリーンな合成データのみ。フェーズ5で今後の課題として言及）
- 実観測データ・実験データへの適用
- 潜在空間での連続拡散（比較検討はしたが採用しない。§2.7 参照）
- 微分方程式・陰関数・ベクトル値関数の発見（明示的なスカラー関数 y = f(x) のみ）
- GPU 分散学習・大規模事前学習（設計上は拡張可能な構造にするが実施しない）
- 外挿性能の系統的評価（評価重点の合意に基づき優先度を下げる。指標としては参考記載のみ）

## 2. 手法の技術的設計

### 2.1 数式のエンコーディング

- **表現**: 式木を**前置記法（ポーランド記法）**で直列化したトークン列。前置記法はアリティが既知なら括弧不要で、パースの成否が構文妥当性の判定になる。
- **語彙**:
  - 二項演算子: `add, sub, mul, div, pow`
  - 単項演算子: `sin, cos, exp, log, sqrt, neg`（`tan, abs` は設定でオン可）
  - 変数: `x0 … x4`（最大5変数）
  - 定数プレースホルダ: `C`（実数定数。値は生成後にフィッティング）
  - 整数リテラル: `1〜5`（`pow` の指数等で頻出する小整数のみ語彙に含める）
  - 特殊トークン: `[PAD], [MASK]`
- **系列長**: 固定長 L（設計値 L=64、プロトタイプ L=32）。パディングで右詰め。
- **可逆性の保証**: `tokenize(parse(s)) == s` の往復一致をユニットテストで担保（フェーズ4必須要件）。

### 2.2 拡散過程: マスク型（吸収状態）離散拡散

D3PM（Austin et al. 2021）の吸収状態（absorbing-state）カーネルを採用する。

- **前向き過程**: 時刻 t で各トークンを独立に確率 β(t) で `[MASK]` に置換。t=T で全トークンがマスクされる。
- **学習目的**: MDLM / MD4 系の知見により、吸収状態モデルの変分下界は「マスクされた位置に対する重み付きクロスエントロピー」に帰着する。実装は「ランダム率でマスク → マスク位置の元トークンを予測」の単純な形になり、CPU プロトタイプに適する。
- **逆過程（サンプリング）**: 全トークン `[MASK]` から開始し、T ステップ（設計値 T=50〜100、プロトタイプ T=20〜50）で徐々にアンマスク。各ステップで確信度の高い位置から確定する confidence-based unmasking を第一候補とする（LLaDA 等で有効性が報告される方式）。
- **選定理由**（§2.7 の代替案比較も参照）:
  1. 先行研究 DDSR（arXiv:2505.24776）・Symbolic-Diffusion（arXiv:2510.07570）がいずれも離散トークン拡散で SR に有効と報告している
  2. 吸収状態は uniform カーネルより実装が単純で学習が安定するという報告が多い
  3. 自己回帰と異なり全トークンを並列にデノイズするため、系列全体のグローバルな整合性を見ながら生成できる（拡散を使う動機そのもの）

### 2.3 データへの条件付け（エンコーダ）

- **入力**: n 点の観測 {(x_i, y_i)}（n は可変、設計値 n≤500、プロトタイプ n=100〜200）。各点は (d+1) 次元ベクトル（d=変数数、不足変数は 0 パディング）。
- **アーキテクチャ**: NeSymReS（Biggio et al. 2021）に倣い **Set Transformer**（induced set attention）で順序不変な要約ベクトル列（m 個の潜在トークン）に圧縮。プロトタイプでは通常の Transformer エンコーダ＋平均プーリングの簡易版から始めてもよい（切替可能に設計）。
- **数値の埋め込み**: 各座標値を IEEE 風の (符号, 仮数, 指数) に分解して埋め込む方式（NeSymReS 系）を設計層とする。プロトタイプは「値の線形層埋め込み＋入力の標準化」で開始し、精度不足なら段階的に高度化。
- **デノイザへの接続**: デコーダ（双方向 Transformer、causal mask なし）の各層から条件ベクトル列へ cross-attention。時刻 t は正弦波埋め込みを加算。

### 2.4 サンプリングと候補選択

1. 1 問題につき k 個（設計値 k=128、プロトタイプ k=16〜32）の候補スケルトンを並列サンプリング
2. 前置記法としてパース不能な候補は**棄却**（棄却率は必ず記録し EXPERIMENTS.md に報告。修復戦略は今後の課題）
3. 各有効候補の `C` を BFGS でフィッティング（マルチスタート 4〜10 回、定義域エラーはペナルティで処理）
4. **選択基準**: 検証点での MSE を主基準とし、同程度なら複雑度（トークン数）が小さい候補を優先（評価重点「簡潔さ」の反映）。`score = MSE + λ * 複雑度` の単純な正則化型も比較する。

### 2.5 定数最適化

- スケルトン中の `C` を c1, c2, … として `scipy.optimize.minimize(method='BFGS')` で MSE 最小化
- 初期値はマルチスタート（標準正規＋固定候補 {1, -1, 0.5, 2}）
- `log`/`sqrt`/`div` の定義域外は大きな有限ペナルティを返して継続（NaN/inf を伝播させない）
- タイムアウト: 1 候補あたり上限秒数を設定（プロトタイプ 2 秒目安）

### 2.6 学習データ（式の事前分布）

Lample & Charton (2019, arXiv:1912.01412) の unary-binary 木サンプリングに準拠する。

- 内部ノード数 b を一様サンプリング（設計値 b≤10、プロトタイプ b≤5）→ ランダム木を生成 → 演算子・葉を事前確率で decorate
- 演算子頻度: `add/mul` を高頻度、`div/pow` を低頻度、単項は中頻度（具体値は EXPERIMENTS.md に記録し、感度を確認）
- 葉: 変数（使用変数はインデックス下位から詰める）、定数 `C`、小整数
- 生成した式は SymPy で simplify・妥当性チェックし、定義域が壊れているもの（全評価点で NaN 等）は棄却
- 評価点 X: 変数ごとに区間（既定 [-4, 4]、`log`/`sqrt` を含む式は正区間）から一様サンプリング
- **重要な設計上の認識**: この事前分布の設計（長さ分布・演算子頻度・定数の範囲）がモデル本体以上に復元率を左右する（フェーズ1で指摘した「未知の未知」#2）。ハイパーパラメータとして外出しし、EXPERIMENTS.md で感度を記録する。

### 2.7 検討した代替案と不採用理由

| 代替案 | 不採用理由 |
|---|---|
| 潜在空間での連続拡散（VAE + ガウス拡散） | 式→潜在ベクトルの AE を別途学習する必要があり部品が増える。離散拡散が SR で機能する先行報告（DDSR, Symbolic-Diffusion）が既にあるため、まず直系を再現する方が検証価値が高い |
| uniform カーネル D3PM | 吸収状態より学習目的が複雑で、MDLM 系の単純化が使えない。Symbolic-Diffusion の採用カーネルの詳細は UNKNOWN（§3）だが、本実装は吸収状態を第一候補とする |
| 自己回帰 Transformer（NeSymReS 型）そのもの | 本プロジェクトの目的（拡散モデルの適用）に反する。ただし比較用の参照実装として意義はあり、余力があれば同一エンコーダ＋自己回帰デコーダを追加ベースラインにする（optional） |
| RL による報酬誘導（DDSR が採用） | 事前学習＋条件付けで まず supervised に閉じる。RL は複雑度が大きく、プロトタイプの範囲外（今後の課題に記載） |

## 4. 評価計画

### 4.1 ベンチマーク

| セット | 内容 | 用途 |
|---|---|---|
| **合成 in-distribution** | 学習と同じ事前分布からの held-out 式 200 本 | 学習が機能しているかの基本検証 |
| **既知式の復元テスト** | `y=2x+1`, `y=x^2+cos(x)` など手選び 10 本（テストにも組込み） | スモークテスト・回帰テスト |
| **Nguyen（10式）** | 標準ベンチマーク。1〜2変数でプロトタイプ規模に合う | 主要評価 |
| **SRSD-Feynman easy サブセット** | 変数数≤3 の式から 10〜20 本抽出 | 物理法則発見という用途への接続。フル 120 式は CPU 予算次第で縮小可 |

### 4.2 指標

1. **記号的一致率（主指標）**: `sympy.simplify(pred - true) == 0` を第一判定、失敗時は `nsimplify`／係数比較による補助判定。**simplify にはタイムアウト（5秒）を設け、タイムアウトは「不一致」に分類**（判定の非決定性はフェーズ1指摘の「未知の未知」#3。判定手続き自体をユニットテストする）。定数フィッティング後の式で判定し、**スケルトン一致率**（定数を除いた構造の一致）も併記
2. **数値精度（副指標）**: テスト点での R²、および R²>0.999 達成率（SRBench 流）
3. **簡潔さ（主指標）**: 復元式の複雑度（式木ノード数）。精度—複雑度のパレート図を作成
4. **無効生成率**: パース不能サンプルの割合（拡散 SR 固有の診断量、「未知の未知」#1）
5. 参考: best-of-k の k に対する性能曲線（k=1, 4, 16, …）

### 4.3 比較対象

| 手法 | 位置づけ |
|---|---|
| 本手法（マスク離散拡散） | 提案 |
| gplearn（GP） | 必須ベースライン |
| 線形回帰 + Lasso（多項式＋初等関数の特徴量辞書上） | 必須ベースライン（「線形手法で十分な問題か」の下限確認） |
| PySR | optional（導入可否をフェーズ4冒頭で判定） |
| 定数なし乱択（事前分布から k 本サンプルして BFGS） | アブレーション: 「拡散＝条件付けの寄与」を分離する対照 |

公平性: 各手法に同一の候補評価予算（またはウォールクロック上限）を与え、条件を EXPERIMENTS.md に記録する。

## 5. 仮定と未知のリスト（間違っていた場合の影響つき）

| # | 仮定 / 未知 | 間違っていた場合の影響 |
|---|---|---|
| A1 | 吸収状態マスク拡散は SR のスケルトン生成に十分（DDSR 等の報告が再現できる） | 復元率がベースラインに遠く及ばない場合、方式再検討（uniform カーネル、自己回帰への後退）。プロジェクトの核心リスク |
| A2 | CPU（数十分〜数時間/回）で 1〜3 変数のプロトタイプが学習収束する | 収束しない場合、語彙・系列長・モデル幅をさらに縮小。最悪「1変数・演算子4種」まで縮めて概念実証に切替 |
| A3 | 学習データの事前分布とベンチマーク（Nguyen 等）の式分布のギャップは致命的でない | ギャップが大きいと in-distribution では動くのにベンチマークで全滅する。事前分布のカバレッジ再設計が必要になり工数増 |
| A4 | パース不能な生成の棄却で十分（修復・文法制約は不要） | 無効率が高すぎる（>50% 等）と実効サンプル数が激減。文法制約付きデコード等の追加実装が必要 |
| A5 | SymPy の等価判定（タイムアウト付き）は評価指標として十分安定 | 判定漏れが多いと記号的一致率が過小評価される。スケルトン一致率・R² を併記することでヘッジ |
| A6 | BFGS マルチスタートで定数フィッティングが実用時間で機能する | 局所解に嵌まる式が多いと「構造は正しいのに不一致」が増える。マルチスタート数の増加または CMA-ES 等への切替 |
| A7 | UNKNOWN: 先行研究（DDSR, Symbolic-Diffusion）の詳細設計（§3 参照） | 本設計が先行研究と意図せず乖離し、比較・再現の主張ができない。レポートでの主張を「同系統の独立実装」に弱める |
| A8 | UNKNOWN: SRSD データセットのライセンスが成果物への同梱を許すか | 同梱不可なら、ダウンロードスクリプトのみ提供に切替（実害小） |
| A9 | UNKNOWN: PySR が本環境に導入可能か | 不可なら gplearn のみで比較（合意済みのフォールバック） |
| A10 | 「クリーンデータのみ」の前提が物理法則発見という用途に対して当面許容される | 実データ適用時はノイズ耐性が必須になる。本プロトタイプの結論がノイズ下で覆る可能性をレポートに明記 |

## 6. プロトタイプの規模目安（参考値）

- モデル: デコーダ 4 層 / d_model 128 / 4 ヘッド、エンコーダ 2〜3 層（合計 ~2M パラメータ級）
