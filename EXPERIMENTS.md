## 1. 設計上の簡略化（原典が確認できなかったもの）

- **損失の重み**: MDLM 系は吸収状態拡散の変分下界がマスク位置クロスエントロピーの
  時刻依存重み付きになることを示すが、原典を閲覧できないため**一様重み**で実装
  （`diffsr/model/diffusion.py`）。生成品質には影響しうる。
- **式サンプリング**: Lample & Charton の厳密な一様 unary-binary 木ではなく、
  予算付き再帰構築の簡易版（`tree.py:sample_tree`）。演算子重みは
  `DEFAULT_OP_WEIGHTS`（add/mul=10, sub=5, sin/cos=3, div/pow=2, exp/log/sqrt/neg=1）。
- **Nguyen の式・区間**: 原典（Uy et al. 2011）未確認。サーベイ再掲の慣用値を使用。
  条件付けデータは各問題の慣用区間から `cfg.n_points` 点（慣用の 20 点より多い。
  エンコーダの入力点数を学習時と揃えるため）。
- **gplearn の protected 演算**: div/log/sqrt はゼロ割り等を握り潰す protected 版
  だが、SymPy 変換では通常演算として扱う（記号的一致がやや甘くなる方向）。

## 2. M4: 無条件拡散の検証

**失敗した試行**: 最初のゲートテストは 2000 式 × 6 エポック（96 ステップ）で
パース成功率 **0.37**（ゲート 0.8 不合格）。学習不足が原因と特定。

学習量を増やした結果（tiny1var 語彙、無条件、8000 式）:

| ステップ | パース成功率 |
|---|---|
| 252 | 0.74 |
| 504 | 0.90 |
| 756 | 0.76 |
| 1008 | 0.93 |
| 1260 | 0.90 |

→ 8000 式 × 20 エポックでゲート **0.8 通過**（テストに反映）。
成功率はステップ間で ±0.1 程度揺れる。マスクスケジュールの調整余地あり（未実施）。

## 3. M5: tiny1var 最小 end-to-end（合格ゲート）

設定: `tiny1var`（1変数、演算子 add/sub/mul/sin、L=16、d_model=64、
20,000 式、8 エポック、シード 0）。学習時間 **約3分**（CPU）、最終 loss 0.605。

ゲート結果（held-out 10 式、k=16、記号的一致判定は定数スナップ込み）: **7/10 合格**

| 正解 | 判定 | 予測 |
|---|---|---|
| 2*x0+1 | OK | 2*x0+1 |
| x0²+x0 | OK | x0*(x0+1) |
| 3*sin(x0) | **NG** | sin(sin(x0)) |
| sin(x0)+x0 | OK | x0+sin(x0) |
| x0³ | OK | x0³ |
| x0²−2 | **NG** | x0²+x0−1.98 |
| 2*sin(x0)+1 | **NG** | sin(sin(x0))+1 |
| sin(2*x0) | OK | sin(2*x0) |
| 5*x0−3 | OK | 5*x0−3.0 |
| x0*sin(x0) | OK | x0*sin(x0) |

観察: 失敗3件はいずれも「正しいスケルトンが k=16 のサンプル中に出なかった」
パターン（BFGS や判定の失敗ではない）。k を増やせば解消する（§4）。

## 4. M6: best-of-k 曲線と λ 感度（tiny1var、ゲート10式）

| k | 復元数 | 平均無効生成率 |
|---|---|---|
| 1 | 1/10 | 0.50 |
| 4 | 4/10 | 0.42 |
| 16 | 7/10 | 0.41 |
| 64 | **9/10** | 0.45 |

- **k が実質性能を決める**（フェーズ1で指摘した「未知の未知」#4 を定量確認）。
  拡散モデル単体の 1 サンプル精度は低く、多様な候補の生成＋BFGS＋選択という
  パイプライン全体で機能する。
- 無効生成率は 0.4〜0.5。棄却で処理できる水準だが高い（リスク A4 は「ギリギリ許容」。
  文法制約付きデコードが今後の課題）。
- λ（複雑度正則化）∈ {0, 1e-4, 1e-2} は復元数・平均複雑度とも**無感度**。
  この規模では MSE だけで候補順位が決まる。
