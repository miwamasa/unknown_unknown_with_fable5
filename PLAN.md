## 1. モジュール構成

パッケージ名: `diffsr`（Diffusion Symbolic Regression）

```
diffsr/
├── expressions/
│   ├── grammar.py       # 演算子定義（名前・アリティ・SymPy/numpy対応）、語彙構築
│   ├── tree.py          # 式木ノード、Lample-Charton流ランダム木サンプリング
│   ├── tokenizer.py     # 式木 ⇔ 前置記法トークン列 ⇔ ID列（往復可逆）
│   └── sympy_bridge.py  # 式木 ⇔ SymPy、等価判定（タイムアウト付き）、複雑度計算
├── data/
│   ├── generator.py     # スケルトン生成→定数割当→X サンプリング→y 評価→フィルタ
│   └── dataset.py       # PyTorch Dataset（オンザフライ生成＋キャッシュ）、collate
├── model/
│   ├── embedding.py     # トークン埋め込み、時刻 t の正弦波埋め込み、数値埋め込み
│   ├── encoder.py       # データエンコーダ（v1: Transformer+平均プーリング / v2: Set Transformer）
│   ├── denoiser.py      # 双方向 Transformer デコーダ（cross-attention 条件付け）
│   └── diffusion.py     # 吸収状態マスク過程、損失、逆過程サンプリングループ
├── fit/
│   └── constants.py     # BFGS マルチスタート定数フィッティング、定義域ペナルティ
├── eval/
│   ├── metrics.py       # 記号的一致率、R²、複雑度、無効生成率
│   ├── benchmarks.py    # Nguyen 10式の定義、SRSD-Feynman サブセットのローダ
│   └── baselines.py     # gplearn / Lasso+特徴量辞書 / 乱択事前分布アブレーション
├── scripts/
│   ├── train.py         # 学習（config 指定）
│   ├── evaluate.py      # 1問題の推論＋定数フィット＋選択
│   └── run_benchmark.py # ベンチマーク一括実行→結果表出力
├── configs/             # dataclass ベースのプリセット（tiny1var / proto3var）
└── tests/               # pytest（モジュール別 + end-to-end）
```

依存: `torch`（CPU版）, `numpy`, `scipy`, `sympy`, `gplearn`, `scikit-learn`, `pytest`。
UNKNOWN: 本環境のプロキシ経由で pip がこれら全てを解決できるか → M0 冒頭で確認、失敗時は代替（バージョン固定緩和等）を報告。

## 3. 最小の動作確認可能な単位（M5 のスコープ定義）

「まず 1 変数の簡単な式で end-to-end を通す」を以下の**tiny1var 構成**として固定する：

- 変数: `x0` のみ
- 演算子: `add, sub, mul, sin`（4種のみ）＋ 定数 `C` ＋ 整数 `1,2,3`
- 系列長 L=16、内部ノード数 b≤4
- モデル: デコーダ2層 / d_model 64 / 2ヘッド、エンコーダは平均プーリング v1（~0.3M パラメータ）
- 学習データ: 2万式（オンザフライ）、CPU 学習 10分以内目標
- 合格条件: `y=2x+1`、`y=x^2+x`（=mul(x,x)+x）、`y=3sin(x)` 級の held-out 10式中 **7式以上を記号的一致で復元**（k=16）

この構成が M5 で通らない限り M7 以降へ進まない（A1・A2 リスクの早期検出ゲート）。

## 4. 検証計画（実装ステップとは分離して記述）

### 4.1 ステップ別検証

| 対象 | 検証方法 |
|---|---|
| M0 | `pytest` が空テストで走る／`pip install` 成功ログ／シード固定で同一 config から同一乱数列が得られることのテスト |
| M1 | **往復一致テスト**: ランダム式 1,000 本で `parse(serialize(tree)) == tree` と `decode(encode(ids)) == ids`。**不正系列テスト**: アリティ不足・過剰トークン列がパース時に明示的例外になる。全語彙トークンが SymPy/numpy 評価に対応している網羅テスト |
| M2 | 固定シードで生成データが完全再現されるテスト。生成 1,000 式の統計検査（長さ分布・演算子頻度・NaN 棄却率が想定レンジ内）。生成式が全て「パース可能・全評価点で有限値」であることの検査 |
| M3 | 既知定数の復元テスト: スケルトン `C*x0+C` とデータ `y=2x+1` から定数 (2, 1) を許容誤差 1e-3 で復元。`log(C*x)` で C<0 初期値でも発散せず終了する定義域テスト。タイムアウト動作テスト |
| M4 | (a) 前向き過程の単体テスト: t=T で全トークンが `[MASK]`、t=0 で原系列一致。(b) 1 バッチへの過学習で損失が ~0 に落ちる sanity check。(c) 無条件サンプル 500 本のパース成功率 ≥80% と、長さ・演算子分布が学習分布に近い（カイ二乗の目視確認、EXPERIMENTS.md に記録） |
| M5 | §3 の合格条件（held-out 10式中7式復元）。学習曲線の記録。**復元テストは pytest に組み込み**（時間がかかるため `@pytest.mark.slow` で分離） |
| M6 | 無効生成率・k に対する復元率カーブ（k=1,4,16）の記録。選択基準（MSE+λ複雑度）の λ を 2〜3 点比較 |
| M7 | in-distribution held-out 200 式での記号的一致率・R²・スケルトン一致率。学習時間の実測（60分以内）。v1/v2 エンコーダの比較 1 点 |
| M8 | ベンチマーク結果表（手法×指標）。全手法に同一ウォールクロック上限を設定した公平性条件の記録。乱択アブレーションとの差＝「条件付き拡散の寄与」の確認 |
| M9 | README の手順を**クリーンなシェルで上から実行して再現**（コピペで通ること）。`pytest` 全通過。docstring の引数・返り値・数式対応の目視レビュー |

### 4.2 横断的検証

- **再現性**: 全スクリプトは `--seed` を持ち、同一シード・同一 config で記号的一致率が完全一致することを M7 で確認（Agent.md 必須要件）
- **等価判定の検証**: `metrics.py` の SymPy 等価判定自体をテスト（等価ペア10組・非等価ペア10組・simplify がタイムアウトする意地悪ペア）。A5 リスクの直接検証
- **失敗の記録**: 各マイルストーンで失敗した試行（発散した学習率、機能しなかった設計）は削除せず EXPERIMENTS.md に残す

## 5. リスクと時間予算

| リスク | 対応 |
|---|---|
| A1（拡散で復元率が出ない） | M5 の合格ゲートで早期検出。不合格なら停止してユーザーに報告し、対応（uniform カーネル試行／スコープ縮小）を協議 |
| A2（CPU で学習が重い） | M4 から常に学習時間を計測。tiny→proto の2段構成で proto を可変にする |
| A4（無効生成率が高い） | M4 の 80% ゲートで検出。不合格時はマスクスケジュール・confidence unmasking の調整を先に試す |
| pip / データセット取得がプロキシで失敗 | M0・M8 冒頭でタイムボックス（各10分）を切って確認し、失敗は即報告（SRSD が取得不能なら Nguyen＋合成のみに縮小） |

推定作業配分（目安）: M0–M3 で全体の 3 割、M4–M6 で 4 割、M7–M9 で 3 割。
